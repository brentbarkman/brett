# Make Inbox a Virtual Concept (Nullable listId) Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `listId` optional on Items so that things without a list are implicitly "in the Inbox" — no more DB-level Inbox list.

**Architecture:** The Prisma `Item.listId` field becomes nullable. The API accepts `listId` as optional on create/update. The `itemToThing()` transform falls back to `"Inbox"` for the `list` display name when `listId` is null. The desktop app removes the auto-create-Inbox-list fallback and simply passes `null` when no list is selected.

**Tech Stack:** Prisma (migration), Hono API, @brett/types, @brett/business, React (desktop app)
